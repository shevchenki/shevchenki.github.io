Progressive Delivery With Argo Rollouts

  • We can cofirm the application before releasing to the endusers
  • We can use Canary Deploy to mesurance User-Exprience

I. Install Argo Rollouts​

  • Quick Start

    kubectl create namespace argo-rollouts
    kubectl delete namespace argocd
    kubectl apply -n argo-rollouts -f https://github.com/argoproj/argo-rollouts/releases/latest/download/install.yaml

  • Confirm running

    k get pods -n argo-rollouts

  • Install kubectl-argo-rollouts on bastion-host

    brew install argoproj/tap/kubectl-argo-rollouts

II. Create Rollout based on BlueGreen Deployment Strategy​

1. Prepare the manifest -> source-code​

  • Convert Deployment to Rollout

    ---
    apiVersion: argoproj.io/v1alpha1
    kind: Rollout
    metadata:
      name: appserver
      namespace: vote-dog-cat-bg
    spec:
      replicas: 1
      revisionHistoryLimit: 2
      selector:
        matchLabels:
          app: appserver
      template:
        metadata:
          labels:
            app: appserver
        spec:
          containers:
          - name: appserver
            image: duynamvnn1208/appserver:v1
            imagePullPolicy: Always
            resources:
              limits:
                memory: "128Mi"
                cpu: "100m"
            ports:
            - containerPort: 8080
          restartPolicy: Always
      strategy:
        blueGreen: 
          activeService: appserver-active
          previewService: appserver-preview
          autoPromotionEnabled: false

  • Change appserver serivce with 2 options (active/preview)

    ---
    apiVersion: v1
    kind: Service
    metadata:
      labels:
        app: appserver
      name: appserver-active
      namespace: vote-dog-cat-bg
    spec:
      type: NodePort
      selector:
        app: appserver
      ports:
      - protocol: TCP
        port: 8080
        targetPort: 8080

    ---
    apiVersion: v1
    kind: Service
    metadata:
      labels:
        app: appserver
      name: appserver-preview
      namespace: vote-dog-cat-bg
    spec:
      type: NodePort
      selector:
        app: appserver
      ports:
      - protocol: TCP
        port: 8081
        targetPort: 8080

  • Change Ingress use alb.ingress.kubernetes.io/group.name

    apiVersion: networking.k8s.io/v1
    kind: Ingress
    metadata:
      name: appserver-active
      namespace: vote-dog-cat-bg
      annotations:
        kubernetes.io/ingress.class: alb
        alb.ingress.kubernetes.io/group.name: appserver
        alb.ingress.kubernetes.io/scheme: internet-facing
        alb.ingress.kubernetes.io/target-type: ip
        alb.ingress.kubernetes.io/security-groups: sandbox_alb_backend
        alb.ingress.kubernetes.io/listen-ports: '[{"HTTP": 80}]'
      labels:
        app: appserver-active
    spec:
      rules:
        - http:
            paths:
              - path: /*
                pathType: ImplementationSpecific
                backend:
                  service:
                    name: appserver-active
                    port:
                      number: 8080

    ---
    apiVersion: networking.k8s.io/v1
    kind: Ingress
    metadata:
      name: appserver-preview
      namespace: vote-dog-cat-bg
      annotations:
        kubernetes.io/ingress.class: alb
        alb.ingress.kubernetes.io/group.name: appserver
        alb.ingress.kubernetes.io/scheme: internet-facing
        alb.ingress.kubernetes.io/target-type: ip
        alb.ingress.kubernetes.io/security-groups: sandbox_alb_backend
        alb.ingress.kubernetes.io/listen-ports: '[{"HTTP": 8080}]'
      labels:
        app: appserver-preview
    spec:
      rules:
        - http:
            paths:
              - path: /*
                pathType: ImplementationSpecific
                backend:
                  service:
                    name: appserver-preview
                    port:
                      number: 8081
